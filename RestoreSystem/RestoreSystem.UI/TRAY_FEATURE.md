# 系統匣常駐功能說明

## ✅ 已實作功能

### 1. **系統匣圖示**
- ✅ 程式啟動後會在右下角系統匣顯示圖示
- ✅ 自動偵測並載入 `Resources\app.ico`（如果存在）
- ✅ 沒有自訂圖示時，使用綠色圓形預設圖示
- ✅ 圖示工具提示顯示「RestoreSystem 還原系統」

### 2. **右鍵選單**
右鍵點擊系統匣圖示會顯示以下選項：
- **顯示主視窗**（粗體，預設動作）
- ────────────
- **服務狀態** - 開啟主視窗並切換到 Dashboard
- **保護設定** - 開啟主視窗並切換到 Protection 面板
- ────────────
- **結束程式** - 關閉應用程式（會確認）

### 3. **視窗行為**
- ✅ **最小化到系統匣**：點擊最小化按鈕時，視窗會隱藏到系統匣
- ✅ **關閉改為最小化**：點擊 ❌ 關閉按鈕時，程式不會結束，而是最小化到系統匣
- ✅ **氣球提示**：首次最小化時會顯示提示訊息
- ✅ **雙擊還原**：雙擊系統匣圖示可重新開啟主視窗

### 4. **退出程式**
- 只能從系統匣右鍵選單選擇「結束程式」
- 會顯示確認對話框：「確定要結束 RestoreSystem 嗎？」
- 確認後才會真正關閉程式並移除系統匣圖示

## 🎨 自訂圖示

### 方式 1：使用現有圖示
1. 準備 `.ico` 格式的圖示檔案（建議包含多種尺寸）
2. 重新命名為 `app.ico`
3. 放在 `RestoreSystem.UI\Resources\` 資料夾
4. 重新編譯專案

### 方式 2：線上產生
1. 前往 https://www.icoconverter.com/
2. 上傳您的 PNG/JPG 圖片（建議 256x256）
3. 選擇輸出尺寸：16x16, 32x32, 48x48, 256x256
4. 下載 ICO 檔案
5. 重新命名為 `app.ico` 並放入 `Resources\` 資料夾

### 方式 3：Visual Studio 內建
1. 在 Visual Studio 中：專案 → 新增項目 → Icon File
2. 使用內建編輯器設計圖示
3. 儲存為 `app.ico` 到 `Resources\` 資料夾

## 📝 技術細節

### 關鍵修改檔案
1. `RestoreSystem.UI\RestoreSystem.UI.csproj`
   - 加入 `<UseWindowsForms>true</UseWindowsForms>`
   - 設定 Resources 資料夾複製到輸出目錄

2. `RestoreSystem.UI\MainWindow.xaml.cs`
   - 加入 `NotifyIcon` 元件
   - 實作系統匣選單
   - 處理視窗最小化/還原邏輯
   - 攔截關閉事件改為最小化

3. `RestoreSystem.UI\App.xaml.cs`
   - 修正命名空間衝突（明確指定 `System.Windows.Application`）

### 命名空間處理
因為同時使用 WPF 和 WinForms，需要明確區分：
```csharp
using WpfBrushes = System.Windows.Media.Brushes;
using MessageBox = System.Windows.MessageBox;
using MessageBoxButton = System.Windows.MessageBoxButton;
using MessageBoxImage = System.Windows.MessageBoxImage;
```

## 🚀 使用體驗

### 正常使用流程
1. 啟動程式 → 顯示登入畫面
2. 登入後可正常使用
3. 點擊最小化 → 隱藏到系統匣（顯示氣球提示）
4. 雙擊系統匣圖示 → 重新開啟視窗
5. 工作結束後：右鍵 → 結束程式 → 確認

### 特殊情況
- **誤關視窗**：程式仍在系統匣運行，雙擊可還原
- **服務監控**：可長期駐留系統匣，需要時再開啟
- **管理操作**：從系統匣選單快速跳轉到特定功能
