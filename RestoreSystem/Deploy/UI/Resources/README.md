# 圖示資源說明

## app.ico
將您的應用程式圖示檔案命名為 `app.ico` 並放在此資料夾中。

### 建議規格：
- **原始圖片尺寸**：**512x512** 或 256x256 像素
- **圖片格式**：PNG（32-bit，含透明背景）
- **最終格式**：ICO（多尺寸合併）
- **必需尺寸**：16x16, 32x32
- **建議尺寸**：16x16, 20x20, 24x24, 32x32, 48x48, 256x256

### 為什麼需要多種尺寸？
- **16x16**：系統匣顯示（100% DPI 螢幕）⭐⭐⭐
- **20x20**：系統匣顯示（125% DPI 螢幕）⭐⭐⭐
- **24x24**：系統匣顯示（150% DPI 螢幕）⭐⭐
- **32x32**：系統匣顯示（200% DPI）、工作列 ⭐⭐⭐
- **48x48**：大圖示模式 ⭐⭐
- **256x256**：Windows Vista+ 高品質顯示 ⭐⭐

### 快速產生方式：
1. 準備 PNG 圖片（**建議 512x512 像素**，透明背景）
2. 前往線上工具：https://www.icoconverter.com/
3. 上傳 PNG 圖片
4. 勾選多尺寸輸出（至少：16, 20, 24, 32, 48, 256）
5. 下載 ICO 檔案並重新命名為 `app.ico`
6. 放在 `RestoreSystem.UI\Resources\` 資料夾中

### 設計建議（RestoreSystem）：
- 主題：盾牌、刷新圖示、保護符號
- 主色：綠色（代表保護啟用）
- 對比：確保在深色/淺色背景都清晰可見
- 簡潔：記住實際顯示只有 16x16 到 32x32 像素

### 目前狀態：
- ✅ 程式會自動偵測 `app.ico` 檔案
- ✅ 如果沒有提供，會自動產生綠色圓形的預設圖示
- ✅ 系統匣圖示會自動使用此圖示

### 部署時注意：
確保 `Resources\app.ico` 被複製到輸出目錄（已在 .csproj 中設定）。

### 測試圖示：
1. 放入 `app.ico` 後重新編譯
2. 執行程式
3. 檢查系統匣圖示是否正確顯示
4. 測試不同 DPI 設定下的效果（Windows 顯示設定 → 縮放比例）
